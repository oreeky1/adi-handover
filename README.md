## ארכיטקטורה — 3 קבצים מחוברים

### 1. `index.html` — תיק החפיפה (ציבורי למחליפה)
התוצר הסופי. מוצג למחליפה, לרוני, ולכל מי שנחשף לתיק.
**כלל ברזל:** התוכן צריך להיראות כאילו עדי בנתה אותו ישירות — בלי שום הפניות לתהליך השאלון.

### 2. `questionnaire.html` — השאלון (פנימי, לעדי בלבד)
מנגנון הזנת תוכן לעדי. כולל:
- **P1–P4 עדיפויות** — סדר מילוי מומלץ
- **באנר "פערים פתוחים"** — הגאפים הספציפיים שהתיק חסר כרגע
- **תגי סטטוס דינמיים** לכל קבוצת שאלות (🔴 ריק / 🟡 חלקי / 🟢 מלא)
- שמירה אוטומטית ב-localStorage + Export ל-JSON
- סיסמה משותפת עם התיק

### 3. `answers.json` — התשובות (מקור אמת)
קובץ JSON עם כל התשובות של עדי. מתעדכן כשעדי לוחצת "שמור ל-GitHub".
זהו המקור ממנו מוזנים עדכוני התיק.

## זרימת עבודה — עדכון התיק

```
עדי ממלאת שאלון  →  answers.json מתעדכן  →  שיחה עם Claude  →  index.html מתעדכן
```

**תהליך מפורט:**
1. עדי ממלאת תשובות חדשות ב-`questionnaire.html` (אוטו-שמירה)
2. לוחצת "שמור ל-GitHub" → `answers.json` מתעדכן
3. אורי פותח שיחה עם Claude ומעלה את `answers.json` המעודכן
4. Claude:
   - משווה תשובות חדשות מול הקיימות
   - מפיק דו"ח מיפוי מקדים (איזה תוכן לאיזה סקשן, לאיזה ממד מ-6)
   - ממתין לאישור
   - מזרים לתיק בשפה עקבית — **6 הממדים לכל תת-סקשן:** מה / איך / מי / מתי / איפה / הערות
5. אורי מבצע `commit + push` לריפו
6. GitHub Pages מפרסם תוך ~30 שניות

## עקרונות עריכה (לצוות העבודה)

1. **6 ממדים עקביים** — כל תוכן ב-role-detail מחולק ל: מה, איך, מי, מתי, איפה, הערות
2. **ידע בעל-פה קודם** — עדיפות גבוהה לשיקול דעת, יחסים, "הגיון בריא"
3. **חוסרים גלויים** — שדה חסר מסומן ב-`יושלם בהמשך`, לא משאירים "שקוף"
4. **ספקים בכרטיסייה** — שם, שירות, איש קשר, חוזה, פקיעה, קריטיות, הערות
5. **לוח שנה ויזואלי** — כל פעילות עונתית משויכת לחודש
6. **עברית RTL** — גופן נקי, קריא
7. **עקביות בין פלטפורמות** — index / questionnaire / answers מסונכרנים
8. **בכל שיחה חדשה:** לקרוא answers.json ו-xlsx לפני עבודה
9. **תיקונים מגיעים לריפו:** אחרי כל שיחה — לוודא commit
10. **מקור מפורש לכל תוכן חדש:** answers.json / xlsx / דברי עדי בשיחה — לא "הגיון" בלבד
11. **ללא שמות פרטיים בכותרות סקשנים** — רק בתוכן. אנשים מתחלפים

## אימיילים — תיקון Cloudflare

GitHub Pages משתמש ב-Cloudflare Email Protection שמערפל אימיילים. כדי להציג אימייל כטקסט רגיל, עוטפים:
```html
<!--email_off-->example@domain.com<!--/email_off-->
```
Cloudflare מזהה את ההערות האלה ומדלג עליהן.
